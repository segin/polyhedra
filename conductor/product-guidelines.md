# Product Guidelines: Polyhedra

## Prose & Documentation Style
- **Technical & Concise:** All documentation, UI labels, and error messages must be clear, direct, and informative. Avoid conversational filler and prioritize high-signal information. Use standard technical terminology for 3D modeling where applicable.

## Branding & Aesthetics
- **Modern & Sleek:** The visual identity should focus on clean lines, high contrast, and a professional aesthetic. A dark-mode-first approach is preferred to emphasize the 3D workspace.
- **Professional & Familiar:** While being modern, the interface should draw inspiration from traditional desktop modeling software (e.g., Blender, AutoCAD) to ensure familiarity for experienced users.
- **Consistent Color Palette:** Establish and adhere to a strict color scheme that reinforces the brand's professional and high-tech feel.
- **Typography-Focused:** Use clear, high-legibility fonts for all textual elements, ensuring readability across various device sizes and resolutions.

## User Experience (UX) Principles
- **Touch-First Design:** Prioritize touch targets and gesture-based navigation (e.g., pinch-to-zoom, two-finger-rotate) for a seamless mobile experience.
- **Efficiency-Driven:** Minimize the number of interactions required for core tasks like adding primitives, transforming objects, and switching between tools.
- **Accessibility (A11y):** Implement accessibility best practices, including high color contrast, screen reader compatibility, and support for alternative input methods where possible.
- **Micro-interactions:** Incorporate subtle animations and visual feedback for user actions (e.g., hover states, button presses, object selection) to create a polished and responsive feel.
